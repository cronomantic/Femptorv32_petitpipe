# Verilator Build Guide

## Overview

FemtoRV32 PetitPipe uses **Verilator 5.0+** for simulation, replacing the legacy Icarus Verilog flow.

**Key Benefits**:
- ✅ C++ compilation → native binary execution (faster than interpreted Verilog)
- ✅ Timing model support (`--timing` flag)
- ✅ Multi-threaded simulation capability
- ✅ Better tracing formats (FST is 4-10× smaller than VCD)
- ✅ Stricter SystemVerilog compliance checking

**Trade-off**: First compilation takes 5-10 minutes (generates ~20K lines of C++), but subsequent executions are very fast.

## Building Simulations

### All Tests (Custom RV32I + Wishbone + Gracilis)

```bash
cd /path/to/Femptorv32_petitpipe

# Compile test programs
make compile

# Run all simulations
make sim              # PetitPipe + Gracilis (9 custom tests each)
make sim-wb           # Wishbone testbench with cache
make sim-gracilis-wb  # Single Gracilis testbench
```

### Official RISC-V Tests

```bash
# Clone official test suite (one-time)
git clone --recurse-submodules https://github.com/riscv/riscv-tests.git \
    third_party/riscv-tests

# Build and run rv32ui (user-mode integer) tests
python3 scripts/run_riscv_tests.py --suites rv32ui

# Build and run all standard suites (rv32ui + rv32mi)
python3 scripts/run_riscv_tests.py --suites rv32ui,rv32mi

# Run specific tests only
python3 scripts/run_riscv_tests.py --suites rv32ui --tests add,addi,and
```

## Compiler Optimizations

### Makefile Variables

```makefile
# C++ compiler optimization level (default: -O2)
make CXXFLAGS="-O3"    # Maximum optimization
make CXXFLAGS="-O1"    # Lower optimization, faster compilation
make CXXFLAGS="-Os"    # Minimize code size
```

### Tracing Format

Two formats supported:

| Format | Pros | Cons | File Size |
|--------|------|------|-----------|
| **VCD** (--trace) | Human-readable text | Slow write, large files | 10-100 MB typical |
| **FST** (--trace-fst) | Binary, 4-10× faster | Less human-readable | 1-10 MB typical |

**Current default**: FST (`--trace-fst`)

To switch to VCD (if needed for legacy tools):
```bash
# Edit Makefile
VLTFLAGS := --cc --exe --build --timing --trace -Wall -Wno-fatal
```

## Generated Artifacts

### Simulation Binaries

Located in `build/sim/`:
- `tb_top` — Basic testbench (stub RTL check)
- `tb_femtorv32_wb` — Wishbone with instruction cache
- `tb_femtorv32_gracilis_wb` — Single-bus Gracilis core

Rebuild rules:
```bash
# Full rebuild
make clean && make sim-wb

# Incremental (recommended)
make sim-wb
```

Located in `build/sim/obj_<testbench>/`:
```
obj_tb_femtorv32_wb/
```

## Trace Files

### FST Format (Recommended)

**Location**: Generated as `dump.fst` in test directory (or via `+vcd=file.fst` flag)

**View/Convert FST**:
```bash
# Install GTKWave (includes FST support)
sudo apt-get install gtkwave

gtkwave dump.fst

# Convert FST → VCD (if integration tool requires it)
verilator_fst2vc dump.fst dump.vcd
```

### Generating Traces

**In testbench, VCD tracing is already enabled** via:
```cpp
// sim_main.cpp (shared harness)
context->traceEverOn(true);
```

This activates `$dumpvars` in Verilog testbench.

**To dump to specific file**:
```bash
./build/sim/tb_femtorv32_wb +vcd=my_trace.vcd
```

## Performance Optimization Tips

### Compilation Speed

1. **Parallel `-j` flag** (Makefile auto-uses available cores)
   ```bash
   # Explicitly set parallel jobs
   make -j$(nproc) sim-wb
   ```

2. **ccache for C++ compilation** (caches object files)
   ```bash
   sudo apt-get install ccache
   # ccache should auto-detect and cache Verilator intermediate objects
   ```

3. **Lower optimization if compiling frequently**
   ```bash
   make CXXFLAGS="-O1" sim-wb    # Faster compile, slightly slower binary
   ```

### Execution Speed

1. **FST vs VCD**: Switch to FST (already default)
   - 4-10× faster trace file writes

2. **Disable tracing if not needed**:
   ```bash
   # Modify Makefile: remove --trace-fst
   VLTFLAGS := --cc --exe --build --timing -Wall -Wno-fatal
   ```

3. **Multi-threaded simulation** (optional, for very large designs):
   ```makefile
   VLTFLAGS := --cc --exe --build --timing --trace-fst --threads 4 ...
   ```

## Troubleshooting

### "Redeclaring std package" Error

**Symptom**:
```
%Error: .../verilated_std.sv:28:9: Redeclaring the 'std' package is not allowed
```

**Cause**: Verilator's SystemVerilog standard library conflicts with user code defining `std` package

**Solution**: Ensure RTL and testbenches don't define a `std` package. This is usually automatic with well-formed Verilog.

### "Cannot find file" Error

**Symptom**:
```
%Error: Cannot find file containing module: /path/to/tb
```

**Cause**: Include path flag malformed (space between `-I` and path)

**Solution**: Use `-Ipath` (no space) in Makefile. Already fixed in provided Makefile.

### Slow Official RISC-V Tests

**Expected behavior**: Each official test takes 5-20 seconds (they run millions of cycles)

**Workaround**: Run subset of tests
```bash
python3 scripts/run_riscv_tests.py --suites rv32ui --tests add,addi
```

## Comparison: Icarus vs Verilator

| Aspect | Icarus | Verilator |
|--------|--------|-----------|
| **Compilation** | <1 second | 2-5 minutes |
| **Per-test execution** | 5-30 seconds | 0.1-2 seconds (after compile) |
| **Tracing overhead** | Minimal | Fast with FST |
| **SystemVerilog support** | Limited | Full |
| **Timing modeling** | No | Yes |
| **Parallel simulation** | No | Yes |
| **Typical total time (9 tests)** | 1-2 minutes | 5 min compile + 1 min tests |
| **Repeated testing workflow** | Better | Much better (reuse binary) |

**Verdict**: Verilator is ~2× slower for one-off test runs, ~5× faster for repeated development workflows.

## Migration Notes

For developers familiar with Icarus Verilog:

### Build Command Change

**Before (Icarus)**:
```bash
iverilog -I tb -o sim.exe tb.v rtl.v
vvp sim.exe +arg=value | tee log.txt
```

**After (Verilator)**:
```bash
verilator --cc --exe --build --timing --trace-fst -I tb -o sim tb.v rtl.v sim_main.cpp
./sim +arg=value | tee log.txt
```

### SystemVerilog Strictness

Verilator enforces:
- No mixing of blocking (`=`) and nonblocking (`<=`) on same signal in sequential logic
- Named blocks required for `disable` statements in tasks
- Proper scoping in modules and interfaces

Check [docs/VERILATOR_COMPLIANCE.md](./VERILATOR_COMPLIANCE.md) for detailed differences.
